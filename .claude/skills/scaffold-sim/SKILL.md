# /scaffold-sim: Simulation Scaffolder

You are a simulation architect. Your job is to take a SimSpec (or a plain-language
description) and produce a fully scaffolded scenario directory — agents defined,
theories wired, parameters set, runner ready. A developer should be able to
`python -m crucible.run scenarios/{slug}` after you're done and see the simulation start.

---

## Parse the request

| Parameter | Description | Example |
|-----------|-------------|---------|
| `spec` | Path to SimSpec JSON or inline description | `forge/specs/pharma.json`, `"US pharma market post-FDA reform"` |
| `slug` | Short scenario identifier (auto-derived if not given) | `pharma-fda`, `hormuz-2` |
| `dry_run` | Print plan but don't write files | `--dry-run` |

If a plain-language description is given instead of a SimSpec path, construct a minimal
SimSpec inline before proceeding (do not call /scaffold-sim recursively — just build it here).

---

## Phase 1: Read SimSpec

If a path is given:
```bash
cat d:/dev/crucible/forge/specs/{spec_file}
```

Extract:
- `scenario_id`, `title`, `domain`
- `actors[]` — each becomes one agent file
- `theories[]` — each becomes one theory module wire
- `parameters{}` — seeded into `params.py`
- `timeframe` — sets tick unit and duration
- `outcome_metrics[]` — determines what the runner measures
- `uncertainties[]` — sets up scenario branching

Derive `slug` from `title` if not provided: lowercase, hyphens, max 20 chars.

---

## Phase 2: Verify prerequisites

Check that the core framework exists before scaffolding:

```bash
ls d:/dev/crucible/core/spec.py 2>/dev/null && echo "CORE_OK" || echo "CORE_MISSING"
ls d:/dev/crucible/core/agents/ 2>/dev/null && echo "AGENTS_OK" || echo "AGENTS_MISSING"
ls d:/dev/crucible/core/theories/ 2>/dev/null && echo "THEORIES_OK" || echo "THEORIES_MISSING"
```

If `CORE_MISSING`: warn the user — "Core framework not yet built. Scaffold will create placeholder
imports. Run Phase 1 (core build) before running the simulation."

Check each theory in `theories[]` exists in the library:
```bash
ls d:/dev/crucible/core/theories/{theory_id}.py 2>/dev/null
```
Flag any missing theories as `THEORY_STUB` — scaffold a stub file with the standard interface.

## Phase 7: Write runner entry point

Write `scenarios/{slug}/run.py`:

```python
"""
Simulation runner: {slug}
Usage: python -m crucible.run scenarios/{slug}
       or: python scenarios/{slug}/run.py
"""
import sys
from pathlib import Path

sys.path.insert(0, str(Path(__file__).parent.parent.parent))

from core.sim_runner import SimRunner
from scenarios.{slug}.params import *
from scenarios.{slug}.theories import get_theories
from scenarios.{slug}.agents import {agent_imports}


def build_agents(env):
    return [
{agent_instantiations}
    ]


if __name__ == "__main__":
    runner = SimRunner(
        scenario_id=SCENARIO_ID,
        title=TITLE,
        tick_unit=TICK_UNIT,
        start_date=START_DATE,
        end_date=END_DATE,
        outcome_metrics=OUTCOME_METRICS,
    )

    env = runner.initialize()
    agents = build_agents(env)
    theories = get_theories(PARAMS)

    runner.run(agents=agents, theories=theories)
```

## Rules

1. **Never overwrite existing files** without asking. If `scenarios/{slug}/` already exists, confirm before proceeding.
2. **Stubs are fine.** A scaffolded sim that doesn't run yet is better than a perfect sim that takes a week to scaffold.
3. **Parameters from SimSpec.** Don't invent parameter values — use what's in the spec. Mark unknowns as `TODO`.
4. **One agent per file.** Never bundle multiple agents into one file.
5. **Flag missing theories.** Never silently skip a theory — always write a stub and note it in the README TODO list.
